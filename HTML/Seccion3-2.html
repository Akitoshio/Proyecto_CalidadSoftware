    Modelos Tradicionales

    Los modelos tradicionales de SDLC ofrecen diferentes enfoques para estructurar proyectos como el Modelo en Cascada, Modelo Espiral, Modelo Incremental, V-Model, Modelo Prototipado y Modelo Iterativo. Son relevantes en entornos donde la predictibilidad es prioritaria, como proyectos gubernamentales o normativos.

    Modelo Cascada

    Descripción

    Es uno de los modelos tradicionales de desarrollo de software, que se caracteriza por su enfoque secuencial y lineal. Por ende en este modelo cada fase debe completarse mas antes de pasar a la siguiente fase, por lo cual normalmente no se regresa a fases anteriores (aunque en la práctica a veces se hacen ajustes).

    Etapas

    • Definición de Requisitos
    • Análisis del Sistema
    • Diseño
    • Codificación
    • Prueba
    • Despliegue
    • Mantenimiento

    Ventajas

    • Claridad de fases
    • Documentación sólida
    • Cumplimiento normativo

    Desventajas

    • Inflexible ante cambios
    • Riesgos detectados tarde

    Casos de uso

    • Proyectos con requisitos estables
    • Entornos altamente regulados

    Modelo Espiral

    Descripción

    Es un modelo de desarrollo de software iterativo y evolutivo, que combina elementos del modelo en cascada con un enfoque principal que es identificar y minimizar riesgos durante todo el desarrollo del proyecto

    Etapas

    • Determinacion de Objetivos y Planificación
    • Analisis de Riesgos
    • Desarrollo o Implementación
    • Evaluacion del cliente

    Ventajas

    • Evalua riesgos explicitamente
    • Permite prototipos
    • Flexible

    Desventajas

    • Complejo de gestionar
    • Costoso

    Casos de uso

    • Proyectos innovadores, criticos y de gran escala

    Modelo Incremental

    Descripción

    Es un enfoque de desarrollo de software en el que el sistema se construye y entrega en partes o incrementos, por lo cual cada incremento agrega nuevas funcionalidades al software existente hasta completar el producto final. Este modelo combina desarrollo por fases con la entrega temprana de partes funcionales, lo que permite a los usuarios probar y usar parcialmente el sistema mientras se sigue desarrollando.

    Etapas

    • Planificación de requisitos
    • Análisis y diseño de cada incremento
    • Implementación o codificación del incremento
    • Pruebas del incremento
    • Entrega del incremento
    • Iteración de nuevos incrementos

    Ventajas

    • feedback temprano, entrega progresiva, menos riesgo de fallo total

    Desventajas

    • Requiere arquitectura modular, riesgo de deuda técnica.

    Casos de uso

    • Sistemas grandes que se construyen por etapas.

    V-Model

    Descripción

    Es un modelo de desarrollo de software basado en la verificación y validación. Por lo cual se divide en dos parte, la parte izquierda representa la definición y diseño, mientras que la parte derecha representa la validación y pruebas.

    Etapas

    • Requisitos del Sistema
    • Requisitos Funcionales y de Software
    • Diseño arquitectonico
    • Diseño detallado
    • Validacion y Pruebas

    Ventajas

    • trazabilidad clara entre requisitos y pruebas

    Desventajas

    • rigidez, requiere pruebas definidas tempranamente.

    Casos de uso

    • sectores donde V&V; son críticos (farmacéutica, aeroespacial)

    Modelo Prototipado

    Descripción

    Es un enfoque del desarrollo de software en el que se construye una versión preliminar o “prototipo” del sistema antes de desarrollar el producto final. Este prototipo sirve para que los usuarios y desarrolladores puedan visualizar, probar y refinar las funcionalidades del software antes de su implementación definitiva, por lo cual su objetivo principal es reducir riesgos y errores, asegurando que el sistema cumpla con las expectativas y necesidades del usuario.

    Etapas

    • Recolección de requisitos
    • Desarrollo del prototipo inicial
    • Evaluación por parte del usuario
    • Refinamiento del prototipo
    • Desarrollo del producto final
    • Mantenimiento

    Ventajas

    • Aclara expectativas, mejora experiencia de usuario.

    Desventajas

    • Puede generar falsas expectativas si prototipo se confunde con producto final.

    Casos de uso

    • Proyectos con requisitos poco claros o alto foco en UX

    Modelo Incremental

    Descripción

    Este modelo es un enfoque de desarrollo de software en el que el sistema se desarrolla en ciclos o iteraciones, por lo cual cada iteración produce una versión parcial del software que se mejora progresivamente hasta alcanzar el producto final. Este modelo a diferencia del modelo en cascada, nos permite revisar y modificar los requisitos durante el desarrollo, lo que hace que sea más flexible y adaptable a cambios.

    Etapas

    • Planificación
    • Analisis y Diseño
    • Implementación/codificacion
    • Pruebas
    • Evaluación/Retroalimentación del usuario
    • Repeticion del ciclo

    Ventajas

    • adaptación y mejora continuaS

    Desventajas

    • riesgo de iteraciones interminables si no se controla.

    Casos de uso

    • proyectos que requieren refinamiento progresivo
