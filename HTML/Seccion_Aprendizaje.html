    Glosario

    Aseguramiento de la calidad (SQA):

    Conjunto de prácticas para garantizar que el proceso de desarrollo cumpla estándares definidos y produzca software confiable.

    Control de calidad (QA Testing):

    Proceso de revisión y prueba del software para detectar defectos en el producto final

    SDLC (Software Development Life Cycle):

    Modelo que define las fases de desarrollo de un software, desde la planificación hasta el mantenimiento.

    Cascada (Waterfall):

    Modelo secuencial donde cada fase debe completarse antes de iniciar la siguiente.

    Espiral:

    Modelo iterativo enfocado en la gestión de riesgos y la creación de prototipos.

    Incremental:

    Modelo que desarrolla el software por partes (incrementos) entregables de forma progresiva.

    V-Model:

    Modelo donde cada etapa de desarrollo tiene una fase de prueba correspondiente.

    Prototipado:

    Creación rápida de versiones de prueba para validar ideas y requisitos antes del desarrollo completo.

    Iterativo:

    Modelo basado en la mejora continua del software a través de versiones sucesivas.

    Kanban:

    Método visual de gestión de flujo de trabajo que limita tareas en curso para mejorar la eficiencia.

    Scrum:

    Metodología ágil basada en sprints cortos, roles definidos y reuniones frecuentes para entregar valor continuo.

    Verificación:

    Proceso de comprobar que el software se ha construido según los requisitos técnicos y de diseño.

    Validación:

    Confirmación de que el producto satisface las necesidades y expectativas del cliente final.

    Pruebas unitarias:

    Verificación de componentes individuales del sistema para garantizar su correcto funcionamiento

    Métricas de calidad:

    Valores numéricos que permiten evaluar el desempeño del software y del proceso (defectos, rendimiento, cobertura de pruebas, etc.).
