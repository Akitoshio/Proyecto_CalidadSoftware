    Técnicas de aseguramiento de la calidad

    Las técnicas aseguran la detección temprana y reducción de errores en diferentes niveles del desarrollo.

    Contenido

    Grafico principal de las tecnicas de calidad de software.

    Pruebas estaticas:

    No requieren ejecutar el código. Incluyen inspecciones de documentos, revisiones de diseño, auditorías de procesos y revisiones por pares (peer review). Son efectivas para identificar errores de lógica o estilo antes de que el software se ejecute.

    Pruebas dinamicas:

    Evalúan el software en ejecución. Incluyen pruebas unitarias (funcionalidad de módulos), de integración (interacción entre módulos), de sistema (funcionamiento global) y de aceptación (criterios del cliente).

    Verificacion y validacion:

    Verificación → "¿Estamos construyendo el producto correctamente?"
    Validación → "¿Estamos construyendo el producto correcto?"

    Gestion de riesgos:

    identificación temprana de puntos críticos en los que el software puede fallar, aplicando medidas de mitigación. Estas técnicas no deben aplicarse de forma aislada, sino en conjunto, para construir un entorno de calidad integral.
